Two carries drive r11: (1) the crop tool was reworked on main (PR #80) to default to non-destructive rectangular slot crops — no more mask paint-out, which was the source of the r1 "figure dissolves into the background" failure; (2) Dimitri wants to test whether the clay conversion step is even needed by meshing straight from the photoreal turnaround.

Goal

Re-run Meshy for Gorgon with the hardened crops, and answer one open question: does the clay step earn its place? We reconstruct from three inputs — the r1 clay sheet, the r2 clay sheet, and the original photoreal turnaround (skipping clay entirely) — and compare the geometry. Clay was adopted to stop the 3D step inventing colours/materials/bases; this round checks whether, with texturing off, going straight from the photoreal sheet is just as good (or better) and saves a whole conversion step.

Technical plan — models, prompts, commands (pipeline detail)

What changed going in (spec / prompt / process deltas)

  • Crops now use the hardened meshy_crops.py from main (PR #80): --crop-mode rectangle (the new default) — split the sheet into rectangular slots and keep pixels UNCHANGED, no background mask / paint-out. This structurally removes the low-contrast dissolution that mask mode caused on the clay sheets (the r9/r10 failure); mask mode is now opt-in for sheets with labels/watermark fragments. Dividers are removed by restitching a divider-free sheet first (as before), then rectangular slots at --scale 1.6.
  • New non-clay arm: the photoreal champion is cropped the same way and fed to Meshy directly. should_texture=False (geometry only) as always — the test is purely whether photoreal shading vs flat clay changes the GEOMETRY.
  • No spec/design change; same meshy-6, 300k triangle, texturing off for all three so the only variable is the input sheet.

Commands (executed this round)

bash cloud/setup.sh
G=projects/cyber-amazons/gorgon
tools/drive_sync.sh pull $G/generated          # clay sheets + photoreal champion

# restitch each 1x4 sheet divider-free, then rectangular (non-destructive) crops
# (r1/r2 reuse the round-10 restitched sheets; the photoreal is restitched here)
for K in r1 r2 gold; do
  python3 tools/meshy_crops.py --sheet <restitched-$K>.png \
    --views front side back three-quarter \
    --crop-mode rectangle --scale 1.6 \
    --out-dir $G/models/round-11/inputs-$K --prefix gorgon-r11-$K-in \
    --composite $G/models/round-11/gorgon-r11-$K-inputs.png
done
# (pre-flight: eyeball each -inputs.png composite before spending credits)

# 3 Meshy runs, one per input; push each mesh to Drive the moment it lands
for K in r1clay r2clay gold; do
  python3 tools/meshy_api.py run --front <$K front> --views <$K side back three-quarter> \
    --out $G/models/round-11/gorgon-r11-$K-m1-raw.stl
  tools/drive_sync.sh push $G/models/round-11
done
# validate/scale to ~37mm, render turntables, comparative review (clay vs non-clay)
