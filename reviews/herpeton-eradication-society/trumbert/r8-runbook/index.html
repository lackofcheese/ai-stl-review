Goal

Find a turnaround recipe that avoids the late-slot ghosting by changing HOW the layout is communicated to flash31, instead of what is drawn inside it. R7 showed the model does its best work when it controls the canvas itself; this round tests progressively lighter-touch layouts — no template at all, a blank sizing canvas, a template pre-filled with the reference, one-angle-at-a-time generation, gradual pre-fill, and regenerating only the ghosted panels of an otherwise-good sheet — to find the cheapest reliable route to four crisp views for the Meshy step.

How the round is judged

Every GENERATED multi-view output gets the turnaround-ghosting v2 screen (pre-filter only — R7 logged sculpted-style false positives, so flags get a human-eye check, and crop audits spot-check unflagged panels), the budgeted blind-detect pass, an output-dimension check — R7's clean template- deviators all measured 2752×1536 while every ghosted 1×4 measured 3512×1184, so dimensions are an instant zero-cost "template ignored" marker — and a fresh-context crop audit with two R8-specific items: layout compliance (did the roll produce the asked-for panels at usable sizes — the no-template arms risk the pre-template failure modes: wrong grids, duplicated or dropped views) and the standing sword-hand digit count. Arm H additionally gets a pixel-diff of the kept panels against the source (drift there is harmless — originals are restitched — but it measures how invasive the repaint was). Arm F/G outputs get a cross-angle consistency check (identity, kit, proportions) since drift is their known risk. Arm N's restitched sheet gets NO fresh audit (Dimitri's call at draft review): the restitch is deterministic and its panels keep their R7 verdicts. A winning recipe = four crisp, correct, consistent views assembled (restitching allowed) reliably enough to feed the R9 Meshy run. Dimitri's eye remains the final gate.

Diagnostic predictions

  • A/B clean → the template image itself is the conditioning artifact (strongest, cheapest recipe). A/B failing on layout compliance (wrong grids, dropped panels — the pre-template failure modes) while C/D succeed → a blank sizing canvas is the sweet spot.
  • E clean where C ghosts → it is specifically an EMPTY template that harms; content in slot 1 re-anchors the model.
  • H producing crisp P3/P4 → the cheapest production path of all: one call repairs any partially-good sheet, and R7's FIXABLE inventory becomes a keeper pool. H failing like R7's F1/F2 → whited panels still inherit the degradation, pointing at position itself rather than content conditioning.
  • F/G consistency drift measured against the R7 multi-view standard decides whether per-angle generation is viable at all.
  • I clean on both pairs → the strongest production recipe: 2 calls per turnaround, nothing ever generated in the degradation zone, better consistency anchoring than F/G. I ghosting in its slot 2 → degradation onset is much earlier than R7's slot-3/4 read and the token-drift hypothesis needs revising.
  • I's cross-PAIR consistency relies on the shared base + held-constant program only (the pairs never see each other; same anchoring R7's full sheets used, and passing views from one lineage already mix freely at the 3D step). If audits show cross-pair build/kit/scale drift, the named reserve fallback is a CHAINED pair-2 re-roll: two inputs, base + the accepted pair-1 sheet, trading independence for consistency.
