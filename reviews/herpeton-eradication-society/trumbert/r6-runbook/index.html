Prior round
R5 (Dimitri): the Base-2 sword fix mis-fired because the prompt said "re-curve", which flash31 read as a *recurve/kukri* blade. Rather than combine every wording idea into one prompt, this round tests the sword-prompting techniques SEPARATELY to learn which one actually works. And r3keeper-rethicken-gptimage2 was the best R5 candidate — take it to a turnaround, but (per the Gorgon experience) do a FAITHFUL full-colour turnaround first, not a monochrome-clay one.

Goal

Two independent things in parallel. (A) Find out *which way of describing the sword* actually gets Trumbert a correct curved scimitar — the last attempts gave a straight broadsword, then a hooked kukri. Instead of piling every trick into one prompt, we run a small A/B/C/D experiment: a plain baseline versus three single-technique prompts, so we can see what's most effective and carry only that forward. (B) Take the strongest R5 render and produce a faithful, full-colour turnaround sheet (four rotating views that keep his exact look) — the multi-view input the 3D step needs.

The choices on the table

Track A — which sword-prompting technique wins. Same protective-interpose Trumbert; we only reshape the blade + fix the grip. Four prompts, each identical except for ONE way of asking for the curve, so the comparison is clean:

VariantThe one technique it addsQuestion it answers
V0 baselinenone — just "make it a curved scimitar, not a broadsword"is any special wording even needed once "re-curve" is gone?
V1 analogy"like a saber / cutlass / talwar"do comparisons to real blades land the shape?
V2 convexnames the geometry: a single convex arc, edge + spine bow the same waydoes precise shape vocabulary work best?
V3 viewer-leftpins the curve toward the viewer's leftdoes anchoring the direction help or just constrain?

Every variant carries the SAME grip fix (one thumb + four fingers, thumb-on-front) as a control, and runs on both flash31 and gptimage2, 2 rolls each. That tells us both the best technique and whether it's model-dependent.

Track B — the turnaround. One approach: three rolls of a faithful, full-colour four-view turntable (front / shield-side / back / sword-side) built from the R5 gold render — same colours, materials and identity, only the camera rotates. (Deliberately NOT a clay-sculpt sheet yet — deferred per the Gorgon experience.)

What we need from you

  1. Which sword technique(s) to keep — after the sweep, a read on whether V0 (nothing special), V1, V2 or V3 gives the most reliable curve, and on which model (flash31 vs gptimage2) should own the sword.
  2. A turnaround pick from the three faithful sheets (or "re-roll"), which becomes the Meshy input next round.

Commands (run after sign-off)

Setup (fresh cloud VM): bash cloud/setup.sh (deps + op CLI + auto-seeds the Gemini balance from tools/gemini-balance.json) then tools/drive_sync.sh pull projects/herpeton-eradication-society/trumbert/generated for the base images + template.

CH=projects/herpeton-eradication-society/trumbert
GEN=$CH/generated
BASE=$GEN/trumbert-r4-interpose-i2i-flash31-1.png

# Track A — 4 prompt variants x (flash31 + gptimage2) x 2 rolls (run each twice)
for v in v0-baseline:v0base v1-analogy:v1analogy v2-convex:v2convex v3-viewerleft:v3vleft; do
  pf=${v%%:*}; stem=${v##*:}
  for roll in 1 2; do
    python3 tools/compare_gen.py --image $BASE \
      --prompt-file $CH/prompts/r6-swordfix-$pf.txt \
      --out-dir $GEN --out-stem trumbert-r6-swordfix-$stem \
      --variants flash31,gptimage2 --lineage $CH/lineage.json
  done
done

# Track B — faithful turnaround, flash31 1x4 (reference + template), 3 rolls
for roll in 1 2 3; do
  python3 tools/gemini_gen.py --model gemini-3.1-flash-image --image-size 2K \
    --image $GEN/trumbert-r5-i2i-r3keeper-rethicken-gptimage2-1.png \
    --image $GEN/trumbert-r6-turnaround-template-1x4.png \
    --prompt-file $CH/prompts/r6-turnaround-faithful.txt \
    --out-dir $GEN --out-stem trumbert-r6-turnaround-flash31 \
    --lineage $CH/lineage.json
done

# Push every landed image to Drive immediately (rule 4), then audit.
tools/drive_sync.sh push $CH/generated
