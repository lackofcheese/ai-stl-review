Goal

Design the head socket — the right-temple interface socket every Firewall FC model wears, which doubles as the gang emblem — as a standalone, mesh-proven asset, and pick one base design to become the team-wide exemplar. This unblocks the socket that character rounds have been deferring (gorgon r5 punted it "until its standalone asset exists"; mace m1 ate the original studs). Scope this round: explore a variety of plate silhouettes, incumbent triangle as the baseline to beat (1a); the socket silhouette IS the whole gang mark, one asset (2a).

Failure modes being pre-empted (from the skill's hard-won rules):

  • micro-hardware vanishes in mesh reconstruction (mace m1 studs) → everything

designed bold + geometric, ≥0.6 mm, one-eye-width plate;

  • micro-hardware dies in novel-view synthesis (0/16 gorgon turnaround panels) →

generate the socket isolated as a temple hero-crop, never inside a character round;

  • male/female term ambiguity ("recessed pins") → a **locked controlled

vocabulary** (spec.md).

What changed going in (spec / prompt / process deltas)

  • New shared sub-project projects/cyber-amazons/head-socket/ — the socket

is team-wide, not per-character (canonical per-project layout, applied to a shared asset).

  • Controlled connector vocabulary locked (spec.md): plate · socket-hole

(recessed) · pin (raised) · collared socket-hole (hybrid). Banned: bore, post, studs, "recessed pins". Verified against connector-industry sources (VIAVI; Wikipedia *Gender of connectors and fasteners*).

  • Default port construction = collared socket-hole (Dimitri, r1) — truest

neural-jack read + collar relief survives the mesh. Pin↔hole is a cheap late flip; not testing both this round.

  • Generation target = temple hero-crop — tight two-panel close-up of only

the right temple on a neutral bald clay head (flat-on to read the glyph + raking 3/4 to read the relief). Maximises pixels on the hardware.

  • Model policy made explicit below (the built-in concept panel string in

compare_gen.py is stale — still lists struck-out models — so pass explicit variant lists, not --variants concept).

Execution

Generation runs after Dimitri signs off on this runbook (runbook-before- generation). Stage 1, all four designs (parallel across the three engines):

H=projects/cyber-amazons/head-socket
G=$H/generated
for P in P1-incumbent P2-nested-triangles P4-delta-chevron P5-round-keyed-jack; do
  STEM=hsocket-r1-$(echo $P | cut -d- -f1)
  python3 tools/compare_gen.py --prompt-file $H/prompts/r1-$P.txt \
    --out-dir $G --out-stem $STEM \
    --variants flash31,gptimage2,grokq --jobs 3 \
    --lineage $H/lineage.json
done

Audit each of the 12 as it lands (fresh-context crop subagent + eye), logging verdicts to reviews.json via tools/review_log.py. Then build the review page (tools/review_page.py --inline → claude.ai artifact for Dimitri; the r1-panel.json config is already stubbed with the asks/next-steps). Stage 2 fires on the design verdict.

Close-out: update rounds.md, push text keepers to the branch, push binaries to Drive (tools/drive_sync.sh push $G).
