How the round is judged

Each returned mesh: immediate Drive push (before anything else), then programmatic validation (watertightness / components / non-manifold edges / Euler via mesh_validate.py, scale note ~24 mm figure), the thin-connection probe (mesh_thin_parts.py, tight settings — scimitar blade/tip, ankh, ears and hair locks are the expected risk features), 4-view turnaround renders plus hand/prop zoom crops, and a fresh-context per-feature COMPARATIVE audit across the five meshes walking the full checklist (scimitar shape+grip, digit counts — thumb+3 expected from the shared r6sword base, shield, chest sunburst, ankh pendant, fur cloak, ears, feet/ground contact, phantom geometry). Verdicts to reviews.jsonl; the r9-panel carries interactive 3D viewers (decimated display proxies over the ~16 MiB artifact cap; exact STLs stay on Drive). Dimitri's live-3D look is the final gate, incl. the pathway comparison against the R1 direct-text baseline mesh.

Technical plan — crops, manifest, commands (pipeline detail)

What changed going in (process deltas)

  • New committed tool tools/turnaround_crops.py — the R8 "N"-cell detected-position crop rule, productionized (the N cell was ad-hoc PIL). Content mask = saturation OR luminance edges (grey steel counts, smooth gradients don't); each axis segmented into exactly R×C figure segments by low-content gap bands, speck segments (detached blade tips) absorbed into the neighbor across the smaller gap; panels exclude the whole gap band and stay clear of detected dark divider columns; --match-height downscales panels to a uniform figure height (used for g14 only, whose P1 runs larger — the N-cell precedent). Two failure modes were caught and fixed during pre-flight on exactly these sheets: a mean-darkness divider test that misread figure-dense columns as dividers, and a fraction-based content profile that clipped thin near-horizontal scimitar blades from two 2×2 shield-side panels (now a solid-run test: any ≥4 px contiguous content run marks a column occupied).
  • Crops before runbook publish (Dimitri, 2026-07-17): deterministic prep that a human can check moved ahead of the publish, so the sign-off covers the exact Meshy inputs. Crops, composites and JSON reports are on Drive under generated/meshy/round-9/; lineage entries trumbert-r9-crops-<cell> record every rect.
  • Meshy manifest (notes/r9-meshy-manifest.json, committed with this runbook): five independent image tasks, meshy-6, raw/no-remesh, so provider task ids are durable and polling is concurrent (meshy_manifest.py); resume with resume --run RUN_ID, never a repeated create.

Commands (run after sign-off)

# setup (cold VM): cloud/setup.sh installs op CLI + seeds balances;
# crops already on Drive: tools/drive_sync.sh pull projects/herpeton-eradication-society/trumbert/generated/meshy/round-9

# 1. five Meshy multi-view runs via the durable manifest coordinator
python3 tools/meshy_manifest.py run \
  --manifest projects/herpeton-eradication-society/trumbert/notes/r9-meshy-manifest.json
# record the printed RUN_ID in rounds.md; after any interruption:
#   python3 tools/meshy_manifest.py resume --run RUN_ID

# 2. per mesh, the moment it lands: Drive push FIRST, then validate + render
# (all commands run from the repo root; P = the project directory)
P=projects/herpeton-eradication-society/trumbert
bash tools/drive_sync.sh push $P/models
python3 tools/mesh_validate.py --stl $P/models/round-9/trumbert-r9-<cell>-m1-raw.stl --label trumbert-r9-<cell>-m1
# scale to the ~24 mm target BEFORE any mm-threshold check (mesh_thin_parts
# runs on the final-scale STL, per its docstring / stl-validation.md)
python3 - <<'PY'
import trimesh
raw = 'projects/herpeton-eradication-society/trumbert/models/round-9/trumbert-r9-<cell>-m1-raw.stl'
m = trimesh.load(raw)
m.apply_scale(24.0 / m.extents[2])
m.export(raw.replace('-raw.stl', '-24mm.stl'))
PY
python3 tools/mesh_thin_parts.py --stl $P/models/round-9/trumbert-r9-<cell>-m1-24mm.stl --pitch 0.28
python3 tools/stl_render.py --stl $P/models/round-9/trumbert-r9-<cell>-m1-24mm.stl \
  --out-dir $P/generated/renders/round-9 --stem trumbert-r9-<cell>-m1 --mode turnaround
# zoom crops of hands/props per mesh as needed: --mode zoom --z-min/--z-max
# fresh-context per-feature comparative audit -> review_log.py -> r9-panel
