Goal

Turn the Round 9 clay turnaround sheets into 3D — feed the clean matte-clay four-view sheets into Meshy's multi-view reconstruction and get back a watertight, print-ready mesh of Gorgon. This is the step that first makes her a real object you can drop into a slicer. The problem it addresses: reconstruction is a stochastic lottery at the feature level (a hand can double, a thin edge can float or vanish) — so we run more than one roll and keep the clean one.

The choices on the table

One path, no competing designs — Meshy multi-view reconstruction of each clay sheet. The variety is deliberate insurance, not a design fork:

What we runWhy
r1 clay → 2 Meshy rollsr1 is the backup keeper; its front-pad "4" had degraded to a faint cross, so it earns a look in 3D too
r2 clay → 2 Meshy rollsr2 is the crisper recommended sheet (legible incised "4"), the primary candidate

Four meshes total. Two rolls per sheet is the standard cheap safety net: surface roughness is stable roll-to-roll, but feature defects are per-roll random, so a second roll usually yields one clean shell. What this round sets up for a decision: which single mesh becomes the print candidate, and it keeps the still-open S7.1 un-mirror call live (the clay was built in the mirrored generation frame; with clay already fixed, the un-mirror is now a late horizontal flip of the STL in the slicer — mechanism still pending your confirmation).

What we need from you

After validation + the comparative review land in rounds.md / the r10-panel review: confirm the keeper mesh (defect-free pick, per clay sheet and overall) to carry to the live-3D gate and test print, and settle the S7.1 un-mirror mechanism (STL-in-slicer horizontal flip vs earlier). Meshes come back at Meshy's raw units; the target is ~32 mm on the longest axis — set your viewer to that to see her life-size.

Cost

Meshy ≈ 20 credits/run × 4 runs = ~80 credits. No image-model spend (clay sheets already exist).

Technical plan — models, prompts, commands (pipeline detail)

What changed going in (spec / prompt / process deltas)

  • Both clay parents, not just r2 (Dimitri's r10 scope). Round 9's own next-step note recommended meshing only the crisper r2 sheet; this round widens to r1 and r2, two rolls each, and picks the defect-free mesh across all four. No spec/design change.
  • No new generation prompt — this is a 3D step, not an image step. Inputs are the existing clay PNGs.
  • Input prep (standard, from the skill's Meshy-crop rules): each clay sheet is a wide 1×4 strip with dark dividers. Divider-trim past the dark dividers and restitch a divider-free sheet FIRST (dividers sit at panel corners and poison meshy_crops.py's corner-based background estimate), then crop the 4 panels in rotational reading order (front → side → back → other side).
  • Frame note (S7.1 — still open): clay was converted in the mirrored generation frame (Gorgon walks viewer-right; pad her-right shoulder). Reconstructing in-frame does not foreclose the team-standard un-mirror; with the clay fixed, that flip is now an STL-in-slicer horizontal flip, pending Dimitri's confirmation.
  • Validation policy (per references/stl-validation.md): watertightness + Euler, scale to ~32 mm longest axis, thin-connection probe (mesh_thin_parts.py tight: pitch ~0.28, erode 1, --gold diff), turntable
    • hand/pad zoom renders, then a fresh-context per-feature COMPARATIVE review vs the photoreal gold (gorgon-r8-B2-turnaround-1x4-flash31) — pre-filter only; the human live-3D gate is the real acceptance.

Commands (run after sign-off)

# setup: deps + secrets (OP_SERVICE_ACCOUNT_TOKEN -> op CLI -> minis-pipeline vault)
bash cloud/setup.sh
G=projects/cyber-amazons/gorgon
GEN=$G/generated
# pull the two clay PNGs from Drive (cloud: Google Drive MCP connector; local: drive_sync)
tools/drive_sync.sh pull $GEN

# per clay sheet: divider-trim + restitch, then crop the 4 panels (reading order)
for R in r1 r2; do
  SHEET=$GEN/gorgon-r9-clay-1x4-$R-flash31-1.png
  python3 tools/meshy_crops.py --sheet $SHEET \
    --views front side back three-quarter \
    --out-dir $G/meshy/round-10/inputs-$R \
    --prefix gorgon-r10-$R-in \
    --composite $G/meshy/round-10/gorgon-r10-$R-inputs.png
done
# (pre-flight: eyeball each -inputs.png composite before spending credits)

# 4 Meshy runs = two rolls per clay sheet
for R in r1 r2; do
  IN=$G/meshy/round-10/inputs-$R
  for ROLL in m1 m2; do
    python3 tools/meshy_api.py run \
      --front $IN/gorgon-r10-$R-in-front.png \
      --views $IN/gorgon-r10-$R-in-side.png $IN/gorgon-r10-$R-in-back.png $IN/gorgon-r10-$R-in-three-quarter.png \
      --out $G/meshy/round-10/gorgon-r10-$R-$ROLL.glb
    # push each mesh to Drive THE MOMENT it lands (before validation)
    tools/drive_sync.sh push $G/meshy/round-10
  done
done
