Goal

R4 got Trumbert thick enough to print but left each strong roll with a single flaw. This round takes the three best starting points, defines the one fix each needs, and runs that fix across all three of our strongest models — so we compare not just three corrected bases but the best *base × editor* pairing, and pick the single strongest corrected image for the turnaround.

The three fixes (one per base)

Each base keeps what it already does well; only its named problem changes. The prompt delta is different for each base — that's why they're separate prompts:

flash31 (R4 img2img roll)Great one-mass hair, great heavy FUR cloak, embedded earsSWORD only — its blade came back a straight broadsword; re-curve it into an unmistakable crescent scimitar (curve prioritized over bulk). Everything else untouched.

The choices on the table

Same locked design, but a full 3×3 sweep: each of the three per-base fixes is run on all three models (riverflow25, flash31, gptimage2). Each model brings a different strength — riverflow the cleanest scimitar, flash31 the best hair and fur cloak, gptimage2 the truest identity — so cross-editing every base surfaces the best pairing (e.g. does gptimage2 hold identity while fixing riverflow's grip? can riverflow put a real curve on flash31's blade?). You pick the single strongest cell (or a hybrid) to carry to the turnaround.

Technical plan — models, prompts, commands (pipeline detail)

What changed going in (spec / prompt / process deltas)

  • No spec rewrite this round. spec.md already demands a *curved* scimitar and a "fingers together on the grip, thumb visible" hold. R5 is an *image-execution* pass forcing those existing rules into the pixels of three chosen bases. Confirmed geometry changes fold into spec.md only after Dimitri signs off on the winning base.
  • Three per-base prompts (each a tight, single-purpose edit — deliberately NOT one shared prompt, because each base needs a different change):
    • prompts/r5-i2i-riverflow-gripfix.txt — KEEP the (already correct) curved blade, hair, cloak, pose *exactly*; change ONLY the right-hand grip to a forward thumb-front / fingers-behind hold.
    • prompts/r5-i2i-flash31-swordfix.txt — KEEP the good hair, fur cloak, ears, grip, identity *exactly*; reshape ONLY the straight broadsword blade into a clearly curved crescent scimitar (solid but curve-first), lowered tip on the ground.
    • prompts/r5-i2i-r3keeper-rethicken.txt — full re-thicken of the clean R3 keeper: one-mass hair, embedded ears, heavy fur cloak, thick curved grounded blade, correct forward grip, plus a push for crisp high-relief detail (sun emblem, ankh, fur, straps) that stays print-thick.
  • Model choice = full 3×3 cross-model sweep (Dimitri, this session): every base's fix runs on riverflow25, flash31 and gptimage2, not just the base's own author model. Rationale: each model has a distinct strength, so the sweep finds the best base×editor pairing rather than assuming the self-edit wins. Known caveats the sweep will expose: gptimage2 tends to re-compose edit sheets (may drift identity/composition), and flash31 was the model that straightened the blade in R4 — but flash31 is now only one of three editors on the sword base, with riverflow25/gptimage2 alongside it as the curve-capable alternatives, so the sweep de-risks that failure instead of betting on it.
  • Scope note: the earlier R5 draft's fresh text→image "prompt-adjust" track stays dropped — R5 is purely the three targeted img2img fixes, now swept across the three models.
  • Audit delta: the crop-audit prompt gains an explicit grip-ORIENTATION check (thumb-on-front / fingers-behind, not merely "a thumb is present") — the R4 crop audits reported "clean grip+thumb" and missed the inversion (verdict-overturns.jsonl trumbert-r4-grip-inverted-crop-audit-miss). Blade shape is audited as "clearly curved scimitar vs straight/broadsword," and each cell is checked for identity/composition drift (the cross-model recompose risk).
