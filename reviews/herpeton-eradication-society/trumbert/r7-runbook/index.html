Detector stack (used to screen every new sheet)

Implementation: tools/turnaround_ghosting.py (turnaround-ghosting/v2). It writes report.json, heatmap.png, and a bounded set of numbered evidence crops, plus a separately named review_candidates list of spatially diverse advisory crops from unflagged panels. The JSON retains raw per-panel candidate density even when a panel stays below the conservative surfacing gate.

DetectorMeasurementPurpose

Use source-native PNGs at a common pixels-per-figure scale. Figure windows mask divider/border pixels; divider windows mask figures crossing the line. Suppress the expected opposite side of a finite-width divider rather than counting its two legitimate edges as a ghost. Armor/shield rims, boot soles, hair strands and specular highlights are explicit false-positive controls.

A window becomes a raw ghosting candidate only when at least two of G1/G2/G3 agree. A non-baseline panel is surfaced only when its candidate density exceeds the noisiest declared clean panel by 0.02; the evidence pack then keeps the eight strongest non-overlapping windows by default. D1 can support a shared-cause hypothesis but cannot fail a figure region. The selected baseline panels are controls and are never surfaced as findings.

E1 is the exception to the usual fixed-baseline audit: reversing the view order moves the historically affected sword/back views into panels 1/2. Audit every E1 sheet twice, once with panels 1/2 as controls and once with panels 3/4 as controls. Treat the union of the two evidence packs and both sets of raw panel densities as the result; neither pass alone may rule out content-following ghosting.

Diagnostic predictions

  • Ghosting follows slots 3/4 in E1 → spatial/canvas-position behaviour.
  • Ghosting follows back/sword into slots 1/2 → view-content complexity.
  • Figure ghosting and divider degradation improve together in E3–E5 → divider conditioning hypothesis gains support. Divider-only improvement means the line is a correlated symptom, not the cause.
  • E2 worsening in its bottom row reproduces Gorgon R8. One clean E2 roll is useful but cannot overturn the historical result.
  • An effect that repeats in the same direction from both bases is more credible than a one-base winner. Base-specific divergence is itself a result and must not be averaged away.
  • E0c improves on E0a with nothing else changed → the anti-ghosting prompt line itself suppresses ghosting; credit it before crediting any layout/divider axis (it would also be the cheapest recipe of all).

Scoring, confirmation and solution ladder

Each sheet receives G1/G2/G3 heatmaps, D1 v2 profiles, a full-sheet blind_detect.py pass (standard and budgeted; run by close_cell, with any blind-vs-crop disagreement a mandatory human look), a fresh-context crop audit of flagged and known regions, normal turnaround checks — including an explicit sword-hand digit COUNT per panel where the hand is visible (both R6 human picks show a thumb + 3 fingers, so inherited wrong counts are recorded as the known base defect, not a new cell regression) — and source-diffs for repair cells. Detector v2 is a pre-filter (70% recall): its verdicts never close a cell on their own, and every keeper candidate goes to Dimitri's eye. A screen winner needs no known-region ghosting failure, no new structural regression, and material improvement over the matching-base FRESH control (E0b/E0c); divider-only gain is insufficient.

Generate two confirmation rolls of the best fresh cell. Promote it only if both beat the matching-base fresh-control distribution and the direction of effect is supported by the other base. Otherwise prefer, in order: isolated P3/P4 repair and re-stitch; compatible clean-panel harvesting and re-stitch; four individually generated views assembled deterministically. Blind sharpening/deconvolution is not preferred because it can sharpen the false contour itself.

Candidate causes under test

  2. Divider/border conditioning bleed.
  3. Back-fur, shield-overlap and sword-view complexity.
  4. Layout-dependent row behaviour, as in Gorgon R8 2×2.
  5. Template resampling of finite-width lines.
  6. Whole-sheet img2img reconstruction tax during self-repair.

# Detector wrapper, defined before first use. Native provider extensions
# vary, so audit every supported image extension of a stem.
audit_stem() {
  local stem=$1 grid=$2 template=$3 divider_width=$4
  local baseline_panels=${5:-1,2}
  local baseline_tag=${baseline_panels//,/-}
  local sheet artifact found=no
  for sheet in "$GEN/$stem"-*.{png,jpg,jpeg,webp}; do
    [ -e "$sheet" ] || continue
    artifact=$(basename "$sheet")
    artifact=${artifact%.*}
    python3 tools/turnaround_ghosting.py "$sheet" \
      --grid "$grid" --baseline-panels "$baseline_panels" --template "$template" \
      --divider-width "$divider_width" --window 96 --stride 32 \
      --out-dir "/tmp/$artifact-ghosting-baseline-$baseline_tag"
    found=yes
  done
  [ "$found" = yes ] || { echo "Missing generated sheets for $stem" >&2; return 1; }
}
